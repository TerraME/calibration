Calibration metrics and algorithms


Version: 0.3

Release: 19 July 2016
Authors: Antonio Oliveira Jr, Pedro R. Andrade, Claus Aranha
URL: https://github.com/pedro-andrade-inpe/calibration
Contact: junior092006@hotmail.com, pedro.andrade@inpe.br, caranha@cs.tsukuba.jp

A package with functions for calibrating spatial models. It contains goodness-of-fit metrics as well as an automatic calibration feature using a genetic algorithm called SaMDE (Self-Adaptive Mutation in the Differential Evolution).

Types

MultipleRuns The Multiple Runs type has various model execution strategies, that can be used by the modeler to compare the results of a model and analyze it's behavior in different scenarios.
SAMDE Type to calibrate a model using genetic algorithm.

Functions

GoodnessOfFit Goodness-of-fit metrics.
Utils Useful functions that are used by MultipleRuns and SAMDE.

Examples

FireAverage Fire in the forest example using multiple runs repeateated strategy.
SIR A Susceptible-Infected-Recovered (SIR) model with a public campaign.
YeastMultipleRuns Basic example for testing MultipleRuns type using a basic Yeast model.
YeastSAMDE Basic example for testing SAMDE type using a Yeast model.
basicMultipleRuns Basic example for testing Calibration type, using factorial strategy.
basicSAMDE Basic example for testing SAMDE type, using a simple equation and variating it's x and y parameters.
broeke An implementation of the model described in ten Broeke, Guus, George van Voorn, and Arend Ligtenberg.
daisy-calibration Daisy world example using multiple Runs factorial strategy.
goodnessOfFit Basic example for testing goodnessofFit type, Example described in the Costanza paper.
infection Infection example using SaMDE, simulates an infection spreading inside a school.
moving-agents Example that uses MultipleRuns to compute wow many cells agents moving randomly can reach.
